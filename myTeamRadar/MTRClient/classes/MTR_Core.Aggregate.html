Class Aggregate

Encapsulates an aggregation of client category options for the myTeamRadar application.

Description

The Aggregate object models the aggregation (creating a union) of different teams (options) within a particular client category. It also allows for filtering an aggregate on cross-category teams. This is equivalent to creating an intersection of teams.

constructor

  • new Aggregate(name?: string, obj?: any): Aggregate

  • Creates an aggregate object.

    Description

    Accepts either a CategoryOption or an array of CategoryOptions to initialize on. If no parameters are passed, the routine exits after setting default initialisations. If 'obj' is passed, it is first determined whether this is a single object or an array. This is determined by checking for the 'CategoryOptionID' field. If this is defined, 'obj' is assumed to be a CategoryOption object.

    If the 'CategoryOptionID' field is undefined, 'obj' is assumed to be an array of CategoryOption objects. In this case, the routine loops through elements of the array, checking that each has a 'CategoryOptionID' field. If this is defined, it is then checked that an element with this ID does not already exist. If it is unique, then it the element is added to the 'options' array.

    Parameters

    • name: string = "empty"

      Name of the Aggregate.

    • obj: any = null

      Either a CategoryOption or array of CategoryOptions.

    Returns Aggregate

FilterSampleList

  • FilterSampleList(userData: UserData[]): void

  • Creates the sample list for calculations.

    Description

    Each element of the userData array has the form:

    {
       id: string;
       CategoryName1: string;
       CategoryName2: string;
       ...
       clarity: number;
       challenge: number;
       flexibility: number;
       autonomy: number;
       recognition: number;
       involvement: number;
       domineering: number;
       detached: number;
       easygoing: number;
       teamwork: number;
    }

    Note that the 'CategoryName' fields are dynamic and depend on the particular client.

    The routine first empties the SampleData array. It then loops through the userData array, checking if the user's category options match anything in the Aggregate's options array (by calling IsInSample. If so, it is further checked that the user matches everything in the filter list (essentially 'AND'ing these conditions). If everything matches, the user is added to the SampleData array.

    Parameters

    • userData: UserData[]

      array of user data

    Returns void
