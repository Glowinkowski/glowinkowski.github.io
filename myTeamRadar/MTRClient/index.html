Overview

Admin application

This is the client-side application for admin use.

On page load, the application calls the initialisation routine init(), which sets up the API endpoints to call for application and round-specific data. These are:

  • /api/AppData
  • /api/roundapi/

(the is provided by the page).

An Application object MyApp is created and two Page objects called Home and Team are added to it. The Start method of the Application object is invoked and the /api/AppData endpoint passed to it, which initialises the application data.

Internally, the Application object calls the Page.Write method of the 'Home' page, which is initialised via the /api/roundapi/ + endpoint.

The rendering function renderClientHome passed to the Home Page object performs the following tasks:

  • Sets up a new CategoryList from the Page object's PageData.
  • Filters the sample lists for each team and calculates the scores.
  • Sets up a new GraphicArea and adds an event handler to call the Team page when the user clicks on the associated TeamGraphic element.
  • Sets up a TeamsLayout object, passing the CategoryList. and GraphicArea to it.
  • Calls the TeamsLayout.SetGraphics method of the TeamsLayout object, which associates a TeamGraphic element with each team.

API endpoints

/api/AppData

The '/api/AppData' endpoint provides the following general application data:

{
   "ClimateText": {
      "clarity": <array-of-text-statements>,
      "challenge": <array-of-text-statements>,
      "flexibility": <array-of-text-statements>,
      "autonomy": <array-of-text-statements>,
      "recognition": <array-of-text-statements>,
      "involvement": <array-of-text-statements>
   },
   "BehaviourText": <array-of-text-statements>,
   "WellbeingText": <array-of-text-statements>,
   "HistoricData": [
      {
         "clarity": <number>,
         "challenge": <number>,
         "flexibility": <number>,
         "autonomy": <number>,
         "recognition": <number>,
         "involvement": <number>
      },
      ...
   ]
}

/api/roundapi/

The roundapi endpoint provides round-specific data:

{
   "RoundID": <number>,
   "ClientCategoryList": [
      {
         "ClientCategoryID": <number>,
         "CategoryName": <CategoryName1>,
         "Options": [
            {
               "CategoryOptionID": <number>,
               "CategoryOptionName": <string>
            },
            ...
         ]
      },
      ...
   ],
   "UserData": [
      {
         "id": <string>,
         "<CategoryName1>": <CategoryOptionID>,
         ...
         "clarity": <number>,
         "challenge": <number>,
         "flexibility": <number>,
         "autonomy": <number>,
         "recognition": <number>,
         "involvement": <number>,
         "domineering": <number>,
         "detached": <number>,
         "easygoing": <number>,
         "teamwork": <number>
      },
      ...
   ]
}
